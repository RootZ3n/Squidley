# Skill: git-commit-messages
## Best Practices for Writing Effective Commit Messages

Effective commit messages should be concise, descriptive, and follow a consistent structure. They typically start with a brief summary of changes (50 characters max), followed by a blank line and a detailed description if necessary.

### Example Format:
```
feat(user): Add user profile picture upload feature
- Allow users to upload their profile pictures.
- Resize images automatically before saving.

fix: Resolve build errors in production environment
- Updated package versions.
- Fixed missing import statements.
```
